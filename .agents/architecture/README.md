# Architecture — Topology & Decisions

Optional layer. Generated by the skill when you opt into "architecture" during setup. Skip for trivial repos.

## What lives here

```
.agents/architecture/
  system.mmd         # C4 Container-level diagram — services, datastores, external APIs
  dataflow.mmd       # representative request/event lifecycle (sequence diagram)
  deployment.mmd     # runtime topology — regions, workers, queues, storage
  decisions/         # Architecture Decision Records (ADRs), one per significant choice
    0000-template.md
    0001-...md
```

All diagrams are Mermaid. Agents parse Mermaid natively; humans render it in any markdown viewer.

## When to update

- **`system.mmd`** — a service is added, removed, or a new external dependency is introduced.
- **`dataflow.mmd`** — the canonical request/event path changes (new middleware, queue, cache).
- **`deployment.mmd`** — runtime topology changes (new region, new compute primitive).
- **`decisions/`** — any non-obvious choice with multi-quarter consequences. Naming: `NNNN-kebab-title.md`. Never rewrite an accepted ADR — supersede it with a new one.

## How agents use this

`global_core.md`'s `<rules id="context-hierarchy">` requires agents to read relevant ADRs before contradicting them, and to read `system.mmd` before changing cross-component contracts. Keep diagrams ≤30 nodes each — split into a second file if larger.
