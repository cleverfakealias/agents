# Windsurf — Best Practices (2026)

Distilled from `docs.windsurf.com/windsurf/cascade/*` and Wave release blogs.

## The four rule triggers

Each `.windsurf/rules/<name>.md` is YAML frontmatter + Markdown body.

```yaml
---
trigger: always_on | manual | model_decision | glob
description: <required for model_decision; ignored for always_on>
globs: <required for glob; comma-separated or single>
---
Body in Markdown. Imperative, short bullets.
```

| Trigger | Behavior |
|---|---|
| `always_on` | Full body injected into Cascade's system prompt every message. |
| `manual` | Not injected. User opts in via `@rule-name` in chat. |
| `model_decision` | Only `description` shown; Cascade pulls full body when relevant. |
| `glob` | Body injected only when Cascade reads/edits a file matching `globs`. |

**Activation mode is the single biggest lever for context-window cost.** Mis-tagging a 12k-char rule as `always_on` wastes budget on every turn.

## `AGENTS.md` — first-class

Per docs.windsurf.com/windsurf/cascade/agents-md:

- **Root** `AGENTS.md` → treated as `always_on` (full body every message).
- **Subdirectory** `AGENTS.md` → auto-promoted to a `glob` rule with pattern `<directory>/**`. Only injected when Cascade touches files there.
- Both `AGENTS.md` and `agents.md` recognized (case-insensitive).
- Fed into the **same engine** as `.windsurf/rules/`. They stack.

**Practical split:**
- `AGENTS.md` — cross-tool portable house rules (read by Codex / Claude / Cursor / Gemini / Windsurf).
- `.windsurf/rules/` — Windsurf-specific overrides, glob scoping, or multi-pattern rules.

## Workflows — `/`-invoked playbooks

`.windsurf/workflows/<name>.md`. Cascade never auto-runs; user types `/<filename>`.

```md
---
description: Cut a release — version bump, changelog, tag, push.
---
1. Verify working tree is clean: run `git status --porcelain` and stop if non-empty.
2. Run `pnpm run ci:verify` (lint + typecheck + test + build). Stop on any failure.
3. Determine bump (patch/minor/major) from commits since last tag — ask if ambiguous.
4. Run `pnpm version <bump>` and update `CHANGELOG.md`.
5. Commit `chore(release): vX.Y.Z`, tag `vX.Y.Z`, push with `git push --follow-tags`.
6. Then call `/address-pr-comments` if a release PR exists.
```

Workflows can call other workflows. **Cap each file at 12k chars.**

## Cascade modes

- **Code** (default) — fully agentic; edits files, runs commands.
- **Plan** — read-mostly; produces a markdown plan with an "Implement" button. Activates `/plan`; plan artifact stored under `~/.codeium/windsurf/brain/` (per-session) and surfaced at `~/.windsurf/plans/`.
- **Ask** — strictly read-only Q&A.

**Switch:** mode toggle below chat input, or `Ctrl+.` / `⌘+.`.

## Memories

Auto-generated by Cascade mid-conversation when it judges a fact worth retaining. **Workspace-scoped, machine-local, never committed.** Located at `~/.codeium/windsurf/memories/`.

**Don't rely on memories for team knowledge** — they don't sync. Write a rule or update `AGENTS.md` instead.

## Global rules

`~/.codeium/windsurf/memories/global_rules.md`. Single file, machine-wide, no frontmatter, always-on. **6k char cap.** Personal house rules (name, comm style, language). Don't put project-specific things here.

## MCP — `~/.codeium/windsurf/mcp_config.json`

Single global file. **No documented project-level MCP config.**

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${env:GH_TOKEN}" }
    },
    "remote-http": {
      "serverUrl": "https://mcp.example.com/mcp",
      "headers": { "Authorization": "Bearer ${env:THING_KEY}" }
    }
  }
}
```

**Hard cap:** Cascade exposes ≤100 tools total across all MCP servers.
