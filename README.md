# agents

A drop-in template for adding AI-agent standards to any repo. Hands every agent (Claude Code, Copilot, Cursor, Windsurf, Gemini CLI, Codex, etc.) the same rules, the same context, and a machine-readable map of your code.

## What you get

- **`ai/global_core.md`** — universal coding rules (types, edit-discipline, security, testing, git). Written once, applied everywhere.
- **`ai/shims/`** — six model-specific override files (Claude, OpenAI, Gemini, Copilot, Cursor, Windsurf). Each ≤50 lines, no rule duplication.
- **`ai/SKILL.md`** — a Claude skill that interactively generates the per-repo files for you. Two paths: auto-explore your codebase or answer 7 guided questions.
- **`AGENTS.md`** (this repo's, at the root) — assembled prompt = `global_core.md` + `project_context.md`. The contract every modern agent reads.
- **`llms.txt`** (this repo's, at the root) — machine-readable index per the [llms.txt spec](https://llmstxt.org).

## Use it on a new repo

```bash
# 1. Copy this folder into your target repo
cp -r ai/ /path/to/your-repo/ai

# 2. Open ai/SKILL.md in a Claude session and invoke the skill
#    Claude detects your stack and generates project_context.md, AGENTS.md, llms.txt

# 3. Before pushing to GitHub, rename for the standard convention
cd /path/to/your-repo && git mv ai/ .ai/
```

## Use it as a reference

Read [`ai/README.md`](ai/README.md) — that's the full documentation: layout, assembly logic, deployment options (submodule vs. CI sync), per-file descriptions, and troubleshooting.

The skill itself is documented in [`ai/SKILL.md`](ai/SKILL.md) (orientation) and [`ai/SKILL-implementation.md`](ai/SKILL-implementation.md) (the execution contract — read this before running the skill).

## Why `ai/` and not `.ai/`?

The folder is `ai/` here because dotfolders are hidden in Obsidian. The standards convention on GitHub is `.ai/` — rename with `git mv ai/ .ai/` before pushing. Details in [`ai/README.md`](ai/README.md#deployment).

## Layout

```
README.md                     ← you are here
AGENTS.md                     ← assembled contract (global_core + project_context)
llms.txt                      ← machine-readable index
ai/
  README.md                   ← full standards docs
  global_core.md              ← universal rules
  project_context.md          ← per-repo specifics (generated by the skill)
  SKILL.md                    ← Claude skill entry point
  SKILL-implementation.md     ← skill execution contract
  shims/                      ← per-model overrides
```
