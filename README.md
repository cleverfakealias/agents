# agents

A drop-in template for adding AI-agent standards to any repo. Hands every agent (Claude Code, Copilot, Cursor, Windsurf, Gemini CLI, Devin, OpenAI Codex, and others) the same rules, the same context, and a machine-readable map of your code.

## What you get

- **`.agents/global_core.md`** — universal coding rules (types, edit-discipline, agentic safety, security, testing, git). Written once, applied everywhere.
- **`.agents/shims/`** — six model-specific override files (Claude, OpenAI, Gemini, Copilot, Cursor, Windsurf). Each ≤50 lines, no rule duplication.
- **`.agents/SKILL.md`** — a Claude skill that interactively generates the per-repo files for you. Two paths: auto-explore your codebase or answer 7 guided questions.
- **`AGENTS.md`** (this repo's, at the root) — assembled prompt = `global_core.md` + `project_context.md`. The contract every modern agent reads.
- **`llms.txt`** (this repo's, at the root) — machine-readable index per the [llms.txt spec](https://llmstxt.org).

**Optional context layers** (opt in per layer during setup):

- **`.agents/architecture/`** — Mermaid topology diagrams (`system.mmd`, `dataflow.mmd`, `deployment.mmd`) + ADRs under `decisions/`. Agents parse Mermaid natively; ADRs prevent re-litigation of past choices.
- **`.agents/intents/`** — Spec-driven work units. Each intent is a markdown file with goal, success criteria, scope, plan. The `## Scope` / `## Out of scope` sections are **binding** for any agent picking up the task — eliminates off-scope drift.
- **Nested `AGENTS.md`** — Per-directory context using the AGENTS.md spec's nearest-ancestor resolution. Each ≤30 lines, local-deltas-only.

## Use it on a new repo

```bash
# 1. Copy this folder into your target repo
cp -r .agents/ /path/to/your-repo/.agents

# 2. Open .agents/SKILL.md in a Claude session and invoke the skill
#    Claude detects your stack and generates project_context.md, AGENTS.md, llms.txt

# 3. Commit — AGENTS.md is the human+agent contract
cd /path/to/your-repo && git add AGENTS.md .agents/ llms.txt && git commit -m "Add agent standards"
```

## Use it as a reference

Read [`.agents/README.md`](.agents/README.md) — that's the full documentation: layout, assembly logic, deployment options (submodule vs. CI sync), per-file descriptions, and troubleshooting.

The skill itself is documented in [`.agents/SKILL.md`](.agents/SKILL.md) (orientation) and [`.agents/SKILL-implementation.md`](.agents/SKILL-implementation.md) (the execution contract — read this before running the skill).

## Why `.agents/`?

The folder is named `.agents/` — a dotfolder that's ignored by most build tools and clearly namespaced as agent context. It's the standard convention across tools that read `AGENTS.md` (Claude Code, Copilot, Cursor, Windsurf, Gemini CLI, Devin, etc.). Details in [`.agents/README.md`](.agents/README.md#deployment).

## Layout

```
README.md                     ← you are here
AGENTS.md                     ← assembled contract (global_core + project_context)
llms.txt                      ← machine-readable index
.agents/
  README.md                   ← full standards docs
  global_core.md              ← universal rules
  project_context.md          ← per-repo specifics (generated by the skill)
  SKILL.md                    ← init skill entry point
  SKILL-implementation.md     ← skill execution contract
  shims/                      ← per-model overrides (Claude, OpenAI, Gemini, Copilot, Cursor, Windsurf)
  skills/
    blueprint/                ← feature planning + intent management skill
```
