# Intents — Spec-Driven Work Units

Optional layer. Generated by the skill when you opt into "intents" during setup.

An **intent** is a tiny spec document written *before* code, carried through the PR, and archived after. It gives agents (and humans) three things in one file:

1. The **goal** and **success criteria** — so the work has a definition of done.
2. A **scope / out-of-scope** boundary — so agents don't drift into adjacent code.
3. A **plan** — numbered steps the agent will execute, in order.

`global_core.md`'s `<rules id="context-hierarchy">` makes the intent file's `## Scope` and `## Out of scope` sections **binding** for any agent working the task. That's the load-bearing property: intents kill off-scope churn at the source.

## Layout

```
.agents/intents/
  intent.template.md           # the template
  open/                        # written, not yet started
    YYYY-MM-DD-<slug>.md
  in-flight/                   # actively being worked
    YYYY-MM-DD-<slug>.md
  done/                        # shipped — kept as an architectural diary
    YYYY-MM-DD-<slug>.md
  abandoned/                   # decided not to do — keeps the rationale
    YYYY-MM-DD-<slug>.md
```

Move the file between subfolders as status changes — no separate status field to keep in sync with the folder.

## Lifecycle

1. **Author** copies `intent.template.md` to `open/YYYY-MM-DD-<slug>.md` and fills in goal, success criteria, scope, plan.
2. **Reviewer** (human or agent) sanity-checks scope and plan. Asks: *Is this the minimal correct change? Are out-of-scope items truly out of scope?*
3. **Move to `in-flight/`** when work starts. Agent links the file in the first commit / PR description.
4. **Move to `done/`** when merged. Don't delete — it's the diary.
5. **Move to `abandoned/`** if dropped. Don't delete — the rationale prevents the same dead-end from being re-explored.

## When NOT to write an intent

- Trivial edits (typo, comment, one-line config). The ceremony costs more than the work.
- Reactive incident work — write a postmortem instead, then a follow-up intent for the durable fix.
